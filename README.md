StoryPointYK — це автономний апаратний аудіоплеєр на базі мікроконтролера ESP32. Пристрій відтворює аудіофайли (MP3/WAV) з SD-карти при спрацьовуванні лазерного датчика відстані (ToF). Керування, моніторинг та конфігурація здійснюються через захищений веб-інтерфейс.

---

## Основні можливості

- **Web-Dashboard:** Адаптивний багатовкладковий інтерфейс для моніторингу показників системи (вільна пам'ять, статус I2S, I2C, Uptime, поточна IP-адреса та mDNS) у реальному часі.
- **Глобальний контроль (Arm/Disarm):** Можливість програмно призупинити роботу системи (відключення реакції датчика та безпечна зупинка відтворення без блокування мікроконтролера).
- **Мережеві можливості:**
  - **mDNS:** Доступ до пристрою за локальним доменним ім'ям (за замовчуванням `http://storypointyk.local`).
  - **Статичний IP:** Підтримка ручного налаштування IP-адреси, шлюзу, маски та DNS.
  - **Wi-Fi Сканер:** Вбудована утиліта асинхронного сканування ефіру для зручного вибору мережі.
- **Планувальник часу:** Налаштування активного вікна роботи за годинами (з підтримкою NTP-синхронізації та ручного встановлення системного часу з ПК).
- **Політика безпеки:**
  - Динамічна зміна облікових даних адміністратора через веб-інтерфейс.
  - Автоматичне відключення власної точки доступу (AP) при успішному підключенні до локального роутера (пристрій стає прихованим у радіоефірі).
  - Захист процесу OTA-оновлення фіксованим інженерним паролем.
- **Керування контентом:** Завантаження та видалення аудіофайлів на SD-карту безпосередньо через браузер. Інтерфейс автоматично розраховує та відображає розмір файлів у мегабайтах.
- **Налаштування середовища:** Регулювання гучності, дистанції спрацювання (в сантиметрах), вибір алгоритмів (послідовно, випадково, фіксований файл).

## Апаратне забезпечення

Перелік необхідних компонентів:
1. ESP32 Development Board
2. MAX98357A (I2S Class-D Amplifier)
3. VL53L0X (Time-of-Flight I2C Sensor)
4. MicroSD Card Adapter (SPI)
5. Динамік 4Ω-8Ω (до 3 Вт)
6. Конденсатор 470 мкФ 16В (Електролітичний) — паралельно лініям живлення MAX98357A.
7. Конденсатор 0.1 мкФ / 104 (Керамічний) — паралельно лініям живлення MicroSD адаптера.

Для компіляції проекту необхідно встановити наступні бібліотеки в Arduino IDE:

1. **ESP8266Audio** (Earle F. Philhower, III) — декодування аудіофайлів.

4. **Adafruit_VL53L0X** (Adafruit) — драйвер лазерного датчика відстані.

## Встановлення та Запуск

### 1. Компіляція та Прошивка
Для забезпечення можливості OTA-оновлень необхідно налаштувати схему розмітки пам'яті.
В Arduino IDE перейдіть до `Tools` -> `Partition Scheme` та оберіть:
* **`Minimal SPIFFS (1.9MB APP with OTA/190KB SPIFFS)`** або **`Default 4MB with spiffs`**.

### 2. Первинна ініціалізація (Captive Portal)
1. Подайте живлення на пристрій. Буде створено відкриту точку доступу **`StoryPointYK`**.
2. Підключіться до мережі та перейдіть за адресою `http://192.168.4.1` (або дочекайтеся Captive Portal).
3. Оберіть режим роботи. У розділі "Розширені налаштування мережі" можна задати власне mDNS ім'я та статичну IP-адресу.
4. Встановіть логін та пароль адміністратора.
5. Збережіть налаштування. Після успішного підключення до зовнішнього роутера власна точка доступу пристрою буде автоматично деактивована.

### 3. Оновлення прошивки (OTA)
1. Згенеруйте скомпільований бінарний файл в Arduino IDE (`Скетч` -> `Експорт бінарного файлу`).
2. У веб-інтерфейсі перейдіть до вкладки "Системні налаштування".
3. Оберіть `.bin` файл у блоці "SYSTEM OTA UPDATE". Актуальна версія прошивки відображається в заголовку блоку.
4. Введіть інженерний пароль для підтвердження прошивки: **`13795OTA`**.
5. Натисніть "Ініціалізувати прошивку" та не відключайте живлення до завершення процесу.

## Вирішення проблем (Troubleshooting)

* **Відсутність звуку:** Перевірте фізичне з'єднання шини I2S. Переконайтеся, що система переведена у стан "Arm" (Запустити систему), а не "Disarm".
* **Переривання відтворення / Помилки ініціалізації SD:** Обов'язково встановіть керамічний блокувальний конденсатор (0.1 мкФ) на лінію живлення SD-модуля.
* **Датчик не ініціює відтворення:** Перевірте налаштування планувальника ("Активне вікно роботи"). Для цілодобового режиму встановіть значення "0" - "0". Синхронізуйте системний час.
* **Не вдається виконати OTA-оновлення:** Перевірте коректність введеного пароля доступу або наявність вільного місця у вибраній `Partition Scheme`.
* **Не працює mDNS:** Переконайтеся, що ваш пристрій (ПК/смартфон) та StoryPointYK знаходяться в одній підмережі і роутер підтримує мультикаст-трафік (Bonjour/mDNS). Якщо mDNS не працює, використовуйте IP-адресу, вказану на екрані роутера.
